# 运行测试文档（Neo4j + LLM）

本文档用于快速联调 STG 离线系统，重点说明数据库连接、LLM 连接和结果输出目录。

## 1. 前置准备

- Python 环境可用（建议 3.11+）。
- 依赖安装完成（至少包含运行本项目需要的依赖）。
- 输入数据存在：`data/less_move.json`。

## 2. Neo4j 连接方式

### 2.1 本机已有 Neo4j

确认以下参数：

- URI：`bolt://localhost:7687`
- 用户名：`neo4j`
- 密码：你的密码

### 2.2 使用 Docker 启动 Neo4j（Windows PowerShell）

```powershell
docker run --name stg-neo4j -p 7474:7474 -p 7687:7687 -e NEO4J_AUTH=neo4j/password -d neo4j:5
```

启动后可在浏览器打开：

- http://localhost:7474

## 3. LLM 连接方式（OpenAI 兼容）

系统使用 OpenAI 兼容接口。

### 3.1 使用 OpenAI 官方

PowerShell 设置环境变量：

```powershell
$env:OPENAI_API_KEY = "你的Key"
```

默认 base url：`https://api.openai.com/v1`

### 3.2 使用其他兼容网关

通过参数覆盖：

- `--llm-base-url`
- `--llm-model`
- `--llm-api-key`

## 4. 运行命令

### 4.1 只跑规则（不连 Neo4j、不启 LLM）

```powershell
python -m stg_system.cli --input data/less_move.json --max-frames 20
```

### 4.2 启用 LLM

```powershell
python -m stg_system.cli --input data/less_move.json --max-frames 20 --enable-llm --llm-model gpt-4o-mini --llm-api-key $env:OPENAI_API_KEY
```

### 4.3 启用 Neo4j

```powershell
python -m stg_system.cli --input data/less_move.json --max-frames 20 --use-neo4j --neo4j-uri bolt://localhost:7687 --neo4j-user neo4j --neo4j-password password
```

### 4.4 同时启用 LLM + Neo4j

```powershell
python -m stg_system.cli --input data/less_move.json  --enable-llm --llm-api-key $env:sk-3086e83fdeba45448623abd593ec14fc --use-neo4j --neo4j-uri bolt://localhost:7687 --neo4j-user neo4j --neo4j-password password --neo4j-database stg.llm --llm-module
```

## 5. 输出目录

系统默认把结果集中输出到 `output` 目录：

- `output/semantic_graph_snapshot.json`
- `output/design_acceptance_report.json`

如果你要指定目录：

```powershell
python -m stg_system.cli --input data/less_move.json --output-dir output
```

## 6. 如何看结果

- 快照文件：看节点、边、frame 映射、边动作事件、owner 决策事件。
- 报告文件：看 requirement_checks、overall、samples。

重点字段：

- `summary.new_edges`
- `summary.duplicate_edges`
- `summary.conflict_edges`
- `summary.owner_assigned`
- `graph_stats.owner_event_stats`

## 7. 常见问题

- Neo4j 连接失败：检查 `--neo4j-uri/--neo4j-user/--neo4j-password`。
- LLM 不生效：检查 `--enable-llm` 与 key 是否有效。
- 无边结果：确认输入关系字段中 `relations` 非空，且对象 idx 可映射。
